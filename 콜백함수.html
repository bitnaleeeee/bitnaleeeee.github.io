비동기처리와 콜백함수

  1. 자바스크립트는 동기적인가 비동기적인가?

    1)자바스크립트는 동기적이다.
    호이스팅이 된 이후부터 코드가 작성된 순서대로 하나하나씩 동기적으로 실행된다.
    *호이스팅이란? var 변수 또는 function declaration이 가장 위로 올라간다.

    console.log(‘1’);
    console.log(‘2’);
    console.log(‘3’);

    차례대로 1,2,3 출력 : 자바스크립트는 작성한 순서대로 실행한다.

    2)자바스크립트는 비동기적이다 : 언제 코드가 실행될지 예측할 수 없다.
    대표적 예시 - setTimeout 함수 (웹 API)
    브라우저에서 제공되어지는 API 로 지정한 시간이 지나면 전달 받은 함수를 호출한다.
    *콜백함수? 파라미터로 브라우저에 전달한 함수를 나중에 호출(callback)하도록 요청 보낸 후 기다리지 않고 바로 다음 코드를 실행해 나감. 해당 요청시간이 지나면 브라우저에서 전달 받았던 함수를 호출(callback)한다. 이렇게 브라우저에 호출(callback)하도록 전달한 함수를 콜백함수라고 한다. 브라우저에 호출을 받은 자바스크립트는 그때 해당 함수(콜백함수)를 실행(출력)한다.

console.log(‘1’);

setTimeout(funciton () {
console.log(‘2’);
}, 1000);

console.log(‘3’);

1, 3, 2 출력

*참고: 위 콜백함수는 아래와 같이 간단하게 작성 가능
setTimeout( ()=>
console.log(‘2’);
,1000);
